Kmom01: Kom igång med teknisk webbdesign

Skriven av Mohamad Alshal, uppdaterad 2025-09-21

Det känns riktigt kul att komma igång med webbteknologier igen! Som någon som redan har programmerat i C++ och Java är det spännande att se hur webbutveckling skiljer sig åt - det är mycket mer visuellt och direkt. Jag har gjort lite grundläggande HTML och CSS tidigare, men den här kursen verkar gå mycket djupare in på teknisk webbdesign och användbarhet.

Att sätta upp utvecklingsmiljön gick smidigt. Jag kör redan VS Code på min Mac och har varit van vid Git sedan tidigare studier, så det var mest att få igång Live Server extensionen och lära mig hur man testar webbsidor lokalt. Det är faktiskt ganska coolt hur snabbt man kan se ändringar direkt i webbläsaren! För browsertestning använder jag främst Safari och Chrome för att se hur sidorna renderas. Developer tools i webbläsaren är fantastiskt användbart för att debugga CSS och se hur elementen påverkar varandra - något jag inte var van vid från mina tidigare programmeringsstudier.

Kurslitteraturen ser riktigt bra ut! Jag har börjat kika på "The Elements of Typographic Style" och "Don't Make Me Think", och det är fascinerande hur mycket tanke som faktiskt läggs bakom bra webbdesign. Som någon som kommer från mer ren programmering är det intressant att lära sig om designprinciper och användarupplevelse. Det som känns lite överväldigande är hur många aspekter det finns att tänka på - det räcker inte med att koden fungerar, utan det måste också vara vackert, användbart, tillgängligt och responsivt. Men det är också det som gör det utmanande och intressant!

Jag har gjort en del grundläggande webbdesign tidigare - mest HTML och CSS för enkla sidor under mina tidigare studier. Men jag har aldrig fokuserat på den tekniska aspekten eller tänkt så mycket på användbarhet och tillgänglighet. De flesta sidorna jag gjort tidigare har varit ganska statiska och inte särskilt genomtänkta designmässigt. Det som är nytt för mig är att tänka på webbdesign som ett tekniskt yrke. Med min bakgrund i C++ och Java har jag mest sett webbutveckling som något "lättare", men nu börjar jag förstå att det finns lika mycket teknik och komplexitet här - bara av en annan typ.

Uppgifterna känns lagom utmanande så här långt. Jag gillar att vi börjar med grunderna i HTML och CSS och bygger uppåt. Den första uppgiften med att skapa me-sidan var rätt straight forward, men jag märkte att jag började tänka mer på semantik och struktur än vad jag gjort tidigare. Det är coolt att få experimentera med olika layouter och se hur CSS påverkar utseendet. Det som var lite klurigt var att få till CSS-layouten riktigt bra, särskilt med responsiv design. Jag är van vid att skriva kod som bara "fungerar", men här måste man också tänka på hur det ser ut på olika skärmar. Jag experimenterade mycket med färggradienter, komplexa hover-effekter och avancerade CSS-animationer - kanske lite väl ambitiöst för första gången, men det var lärorikt!

Min största "Today I Learned" för det här kursmomentet är nog hur mycket tanke som faktiskt ska läggas bakom varje designbeslut. När jag programmerade C++ eller Java tänkte jag mest "fungerar koden?", men nu måste jag fråga "hur upplever användaren det här?" Det är en helt annan mindset. Jag lärde mig också mycket om semantisk HTML - att använda rätt element för rätt syfte, inte bara för att det ser ut som jag vill. Som person som brinner för att skriva ren kod uppskattar jag verkligen den här aspekten av webbutveckling. Jag insåg också att jag kanske gick lite överboard med de komplexa animationerna och effekterna - det är lätt att fastna i att experimentera med allt som är möjligt istället för att fokusera på vad som faktiskt behövs. Men det var ändå värdefullt att testa gränserna för att förstå vad som är möjligt.

Kmom03

Skriven av Mohamad Alshal, uppdaterad 2025-09-19

För min one-page webbplats valde jag temat GameHub, en gaming-community webbplats. Gaming är något jag har stor passion för, så det kändes naturligt att bygga en sida kring något jag verkligen brinner för. Webbplatsen handlar om att samla gamers, erbjuda recensioner, guider och skapa en community för spelare. När det kommer till färgpaletten använder jag en lila/purple tema med #6b46c1 som huvudfärg i headern, och sedan olika nyanser av lila genom hela sidan - från ljusa lila bakgrunder till mörkare lila för footern. Detta skapar en harmonisk färgskala som känns sammanhållen.

För att hitta bilder och material använde jag mig av Xbox Game Pass, som jag anpassade till gaming-temat. Texten skrev jag själv och fokuserade på innehåll som kändes relevant för en gaming-webbplats, som information om spelgenrer, community-funktioner och gaming-zoner. Min huvudsakliga inspiration kom från referenswebbplatsen som visades, där jag kunde se hur en professionell one-page layout fungerar med hero-image, tvåkolumns-innehåll och kortsektioner.

Den största utmaningen var att balansera färgtemat. Först blev det alldeles för mycket lila överallt vilket såg ut som ett filter över hela sidan. Jag fick lära mig att använda färg mer sparsamt och strategiskt. En annan utmaning var att få mobilmenyn att fungera ordentligt - jag behövde justera media queries och förstå hur JavaScript och CSS arbetar tillsammans för responsiv design. Tidspressen var också en faktor som påverkade mina beslut. Jag övervägde att lägga till smooth scrolling-animationer när man klickar på navigationen så att sidan skulle glida upp till rätt sektion istället för att bara hoppa dit direkt, men när jag kollade uppgiftsbeskrivningen mer noggrant stod det inte som ett krav så jag valde att ta bort det för att fokusera på grundfunktionaliteten istället.

För någon som är nybörjare på CSS skulle jag säga att det viktigaste är att börja med grunderna och inte försöka göra för komplicerat från början. Lära sig margin och padding först, sedan display och position. Flexbox och Grid kommer också vara användbara tidigt. Det hjälper att testa en sak i taget och se vad som händer. Utvecklarverktygens inspektör är också väldigt bra för att förstå vad som pågår.

Jämfört med kmom01 har jag utvecklats en hel del. I kmom01 experimenterade jag mycket med olika CSS-tekniker och animationer, vilket var bra för att praktisera den typen av kod och förstå vad som var möjligt. Men för kmom03 valde jag medvetet en enklare väg. Min kod är nu mycket enklare och mer läsbar. Jag har bättre färghantering, från den beige/orange paletten i kmom01 till en sammanhållen lila-tema. Layout-strukturen är också förbättrad, från det komplexa profile-layout systemet till ett enklare grid-system.

Den begränsade tiden påverkade också mina designbeslut. Jag tog bort de komplexa hover-effekterna som fanns i kmom01 eftersom jag inte hade tid att implementera dem på ett bra sätt och de kändes inte nödvändiga för grundfunktionaliteten. JavaScript är också förenklat från komplex hamburger-animation till basic toggle-funktionalitet. Jag minskade användningen av CSS-transitions och avancerade selectors, vilket faktiskt gjorde koden mer läsbar.

Min största lärdom för detta kmom är vikten av att hålla det enkelt och fokusera på det som verkligen behövs. Jag lärde mig att det är bättre att ha en färg som används rätt än att försöka använda den överallt. Jag förstod också värdet av att prioritera grundfunktionalitet över fancy animationer, särskilt när tiden är begränsad. Detta gjorde koden mycket lättare att arbeta med än kmom01, där jag hade gjort det mer komplicerat än nödvändigt.
